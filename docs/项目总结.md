## 一、项目概览

**WutheringWavesDmgCalc** 是《鸣潮》（Wuthering Waves）的伤害计算器桌面应用，基于 PyQt6 开发。项目从 2026 年 5 月 7 日开始构思，5 月 12 日正式动工，6 月 12 日发布 v1.0 正式版，历时约一个月。

### 功能总览

- **完整伤害计算引擎** — 9 大乘区（基础/加成/加深/暴击/倍率/防御/抗性/独立乘区/全伤害），damage_calc.py 为唯一真相来源
- **贡献者系统** — 预设 author 字段 + 欢迎页贡献者名单弹窗 + CONTRIBUTORS.md 自动维护
- **OCR 图文识别** — 支持声骸属性截图识别、技能倍率截图识别（PP-OCRv5 + ONNX Runtime，支持跨行倍率合并与中文字碎片重组（技能分类行不参与合并，prev_line过滤治疗/纯标点行），子分类OCR纠错（暴击→重击，仅纯文字行））
- **预设系统** — 角色/武器/声骸套装/角色增益四类预设，含独立计算引擎和关键词关联
- **官方预设同步** — 从 GitHub 一键拉取最新预设，多源 fallback（直连 + 镜像 + CDN），国内无需 VPN
- **暗色/亮色双主题** — 全界面自适应配色
- **存档系统** — 完整 JSON 存档，支持保存/加载/快速保存
- **错误自动报告** — 崩溃时自动启动 ErrorViewer 弹窗，错误日志持久化
- **自动化测试** — pytest 187 个用例，2.9 秒跑完
- **Git 版本管理** — 内置版本快照，可随时回退
- **辅助工具** — Git 代理管理工具 + 官方预设上传工具（均已打包为独立 exe）

## 三、数据流

### 核心公式

```
最终伤害 = 基础乘区 × 加成乘区 × 加深乘区 × 暴击乘区 × 倍率乘区
          × 防御减伤 × 抗性乘区 × 独立乘区 × 全伤害加成
```

### 数据链路（12 条）

1. **角色基础值 → 最终伤害** — 基础攻击/生命/防御注入计算公式
2. **综合填写（常驻+触发）→ 最终伤害** — 效果条目按乘区分类路由
3. **声骸 → 最终伤害** — 主/固/副词条按名称分类到各乘区
4. **共鸣链增益 → 综合填写 + 关键词关联** — 6 链卡片式管理，启用联动
5. **关键词关联 → 倍率设置 + 最终伤害** — 关键词交集匹配，「倍率增加/倍率提升」词条自动注入倍率乘区（相加/独立乘算），倍率设置表格 8 列实时镜像显示，副名称双向互通
6. **敌人防御减伤 → 最终伤害** — 等级差公式（含无视防御、截断逻辑）
7. **敌人抗性 → 最终伤害** — 正负抗性 clamp 处理
8. **独立乘区 → 最终伤害** — 组内加法 × 组间乘法
9. **ResultPage → 计算结果 → 结果列表** — 多方案对比
10. **上游任意变更 → 全局回调链** — 300ms 防抖自动重算
11. **总结页反向编辑 → 上游同步** — 副名称双向同步
12. **存档保存/加载** — 完整状态序列化

### 打包步骤

```bash

pip install pyinstaller

# 2. 打包全部（主程序 + 2 个工具，输出到 packaging/dist/）
cd packaging
rm -rf build dist __pycache__
pyinstaller --clean --noconfirm WWDmgCalc.spec
pyinstaller --clean --noconfirm git_proxy_manager.spec
pyinstaller --clean --noconfirm preset_uploader.spec

# 3. 重组结构（移动到项目根 dist/）

```

### 打包注意事项

1. **`save/` 仅保留空目录** — 不复制任何存档文件，开发机的测试存档不会进入发布包
2. **`presets/user/` 仅保留空子目录** — character/ character_buff/ echo_set/ weapon/ 四个子目录均为空，不包含任何 JSON 文件
3. **`presets/official/` 复制全部文件** — 官方预设 JSON + manifest.json 正常复制
4. **每次打包前必须检查** — 确认 save/ 和 presets/user/ 没有不该出现的文件
5. **spec 路径自动定位** — `WWDmgCalc.spec` 开头用 `sys.path` 自动搜索 `rapidocr_onnxruntime` 路径，不应硬编码绝对路径。换电脑/GitHub Actions 后打包不会因路径缺失而失败
6. **`.gitignore` 无重复项** — 去重后 43 行，每条忽略规则只出现一次。修改忽略规则时确保不引入重复行，否则可能导致规则失效
7. **`after_build.py` 清理测试文件** — 步骤 2（移动主程序后）自动删除 `_internal/error_handler/test_crash.py`，防止崩溃测试代码进入用户发布包
8. **快捷方式无绝对路径** — `create_shortcut` 不设置 `WorkingDirectory`（否则用户机器上报错"文件夹无效"），仅保留 `TargetPath + Description + IconLocation`
9. **文件夹图标** — `after_build.py` 复制 `ico/icon.ico` 并生成 `desktop.ini`，文件夹在资源管理器中显示自定义图标

### 路径说明（打包后）

| 路径类型 | 取值 | 说明 |
|----------|------|------|
| 用户数据 | `_APP_DIR` = exe 的父级父级 | 存 save/ config/ presets/ |
| 只读数据 | `_DATA_DIR` = `sys._MEIPASS`（即 _internal/） | 存 manual/ error_handler/ models/ |

| 精度清洗 | `_round_floats()` — 保存/导出前递归 round(4) | 消除 IEEE 754 垃圾尾数，恢复用户输入的真实值（536.799...→536.8） |

### 隐藏/锁定状态 Key 规范（4 元组）

全局状态 `HIDDEN_ITEMS` 和 `LOCKED_SUMMARY_ITEMS` 使用统一的 4 元组 key：

```
(name, source, nav_key, seq_label)
│       │       │         └─ 序列号，如 "常驻2" / "触发5"
│       │       └─ 页面标识，如 "combined_perm" / "combined_trigger"
│       └─ 来源，如 "声骸" / "共鸣链效果" / "keyword_assoc"
└─ 词条名称，如 "攻击力加成"
```

涉及的读写位置：
- `WWDmgCalc.py`: CombinedEntryPage 的行创建/隐藏按钮/锁定按钮/删除
- `summary_pages.py`: _fill_source_table / _toggle_hide_item / _toggle_summary_lock / _delete_summary_item
- 计算过滤: `ResultListPage.recalc()` / `ResultPage.compute()` / 关键词关联注入

**所有位置必须使用 4 元组**，不得退化为 3 元组。修改相关代码后确保 `_validate_items` 断言无异常。

## 九、Bug 分类知识库（2026-06-16 总结）

所有 bug 可归入以下 5 类。排查时优先检查前三项（对象、格式、时序），80% 的难以定位的 bug 属于这些类别。

### 1. 对象同一性（Identity）— 最难察觉

| 特征 | 代码表象 | 本项目典型案例 |
|------|---------|-------------|
| 两个变量"看起来是同一个东西"但实际不是 | `set()` 在两个地方各建了一份 | **HIDDEN_ITEMS 两个 set** — toggle 写入 A，compute 读 B，永远对不上 |
| 注入失败、模块缓存分裂、import 路径不同 | 不同入口 import 同一模块，Python 可能创建两份 | summary_pages 的注入链路看似传递了引用，实际被模块缓存分叉 |

**排查方法**: `print(id(x))` 看是不是同一个对象。如果是 `set/dict/list` 跨文件共享→抽到独立模块再导出（本项目已通过 `shared_state.py` 解决）。

**防范原则**: 全局可变状态（set/dict/list）跨文件共享时，从单一模块导出，不依赖注入传递。

---

### 2. 格式/结构（Format）— 最容易犯

| 特征 | 代码表象 | 本项目典型案例 |
|------|---------|-------------|
| 元组长度不同 | `(name, nav_key, seq)` 3 元组 vs `(name, source, nav_key, seq)` 4 元组 | HIDDEN_ITEMS/LOCKED 读写键格式不一致，10+ 处检查失效 |
| 字段含义不同 | source 在一处是行级来源、另一处是页面标签 | 综合填写点"查看总结"跳转失败 |
| 缩进错位 | 类定义被打断、方法被踢出类外 | `_round_floats` 放在模块级，`save_preset` 等 6 个方法被踢出 `PresetManager` |
| 序列号前缀缺失/重复 | 纯数字 `1/2/3` vs `常驻1/触发2` vs `触发触发3` | _collect_all_items 二次加前缀 |

**排查方法**: 打印 `type(x)`, `len(x)`, 看元组结构和字符串内容是否与预期一致。

**防范原则**: `_validate_items` 断言（`_collect_all_items` 末尾）检查每条元组长度 ≥ 6。

---

### 3. 时序（Timing）— 最隐蔽

| 特征 | 代码表象 | 本项目典型案例 |
|------|---------|-------------|
| 数据在 A 时刻改了，B 时刻才读写，中间被覆盖 | 回调链顺序错乱 | 预设加载后 `_on_change_cb` 触发重建表格 → 覆盖刚点的隐藏 |
| Qt 事件循环排队、QTimer 延迟 | `QTimer.singleShot(150, ...)` 时表格未渲染完成 | 监督器 150ms 后找不到按钮 |

**排查方法**: 在每步加 `print` 看执行顺序。特别注意 `_on_change_cb` 回调链：`sp.recalc()` → `_fill_source_table` → 重建按钮 → 覆盖旧状态。

**防范原则**: 任何操作完成后调 `_resequence()` 统一重排（本项目已在 `apply_preset` 末尾和 `_sync_chain_to_pages` 末尾加入）。

---

### 4. 生命周期（Lifetime）— 最防不胜防

| 特征 | 代码表象 | 本项目典型案例 |
|------|---------|-------------|
| 缓存过期：存了旧值，下次读取时没刷新 | `_last_computed["process_html"]` 存的是标签旧文字 | 隐藏后 compute 更新了 zones 但 HTML 还是旧的 |
| 引用被重建/替换 | `_fill_source_table` 建了新按钮，toggle 还在改旧按钮 | summary_pages 的同步块访问已被删除的 `hide_btn`/`lock_btn` 键 → KeyError |
| 对象被 GC/deleteLater 干掉 | 声骸页删了表格但回调还在跑 | 删行后同步块访问 `rd['hide_btn']` → 崩溃 |

**排查方法**: 怀疑缓存时直接看源头数据长什么样。检查 `KeyError` 是否来自访问已不存在的键。

**防范原则**: 从 CombinedEntryPage 移除按钮时同步清理所有引用方（`preset_manager` / `summary_pages`）。

---

### 5. 逻辑（Logic）— 最常见的

## 九、开发注意事项

### HIDDEN_ITEMS（隐藏功能）架构
- **键格式**：四元组 `(name, source, nav_key, seq_label)`，全代码库统一
- **三层数据流**：`_collect_all_items` 全量收集（不过滤）→ `compute()`/`_recalc_one` 计算层过滤 → KW 注入层层过滤
- **原则**：`_collect_all_items` 永不过滤 HIDDEN_ITEMS，表格需要看到全部条目；计算层必须检查
- 详见 `版本更新总结.md`「HIDDEN_ITEMS 彻底修复」章节

### 错误处理
- 所有弹窗统一 `_center_window()` 居中屏幕
- 崩溃捕获自动写入 error_log.json + 启动 ErrorViewer
- 预设同步失败写入错误日志，侧边栏可查看

### 声骸阶段约束
- 应用预设弹窗：勾选各套装的各阶段，总件数 ≤5，超标禁用确定按钮
- 编辑已有预设：load_data 时 blockSignals 防止信号冲突导致 UI 错乱

### 退出保存
- 关闭窗口时弹出「保存并退出/不保存/取消」对话框
- X 按钮 = 取消（隐藏 Cancel 按钮绑定关闭事件）

### 文件名规范
- 存档名和预设名禁止包含 `\/:*?"<>|` 等非法字符
- 输入时弹窗提示，不静默删除

### 倍率系统架构（v1.0 重构后）
- **唯一来源**：关键词关联页 → `_gather_mult_data()` → `compute()` / `_recalc_one`
- **倍率设置表格**：8列表格（名称/副名称/序列号/数值/取值/来源/关键词关联/操作），照搬关键词关联界面结构
- **倍率增加**：所有 name 含「倍率增加」的关键词条目值累加到 `mult_inc`，与基础倍率相加
- **倍率提升**：所有 name 含「倍率提升」的关键词条目值各自独立乘算 `(1 + v/100)`
- **关键词强制要求**：条目必须有关键词 AND 与卡片关键词有交集，否则不显示、不计算
- **HIDDEN_ITEMS 联动**：操作列「隐藏」按钮切换全局隐藏，`_gather_mult_data` 过滤排除
- **双向互通**：副名称编辑实时映射回关键词关联对应行；来源按钮点击跳转+滚动高亮
- **折叠展开**：倍率增加/倍率提升表格可折叠，默认展开，折叠后框架自适应缩小
- **tooltip 唯一性**：sub_map 键使用 seq_label（全局唯一的序列号），杜绝同名同源条目 tooltip 互相覆盖
- **数值总结筛选**：加成乘区/加深乘区表格上方新增筛选芯片行，按元素属性/技能类型/效应类型快速过滤

### 计算分离逻辑
- `def_multiplier` = 仅通用词条的防御乘区（存旧兼容）
- `_skill_zones = {"普攻": 0.54, "重击": 0.54, ...}` = 通用+对应技能专用
- `get_defense_zone(skill_type=None)` → 根据技能类型返回对应值
- ResultPage/ResultListPage 根据卡片 `skill` 属性调用 `get_defense_zone(skill)` 获取个性化防御乘区
- 防御页视角切换按钮调用 `_on_view_skill(skill)` → `_refresh_result_labels()` 实时更新计算结果框

### 时效筛选
- 每表独立 `_timing_filters[key]` ∈ {"全部","常驻","触发"}
- 芯片互斥选中，`_refill_one(table, items, key)` 根据 `_matches_timing(eff_type, key)` 过滤行
- 表格始终全量填充，仅显示过滤 — 取消勾选的词条通过 `_disabled_items` 集合从计算中排除但表格保留

### 双向视角跳转
- 综合填写「查看总结」→ `_navigate_to_summary()` → `is_defense_item(name)` 判定 → `enemy_defense` 导航
- 导航后 `highlight_item(name, source, nav_key, seq_label)` → 遍历 7 表按序列号匹配 → 平滑滚动（15帧×16ms）→ 标准黄色叠层（_place_highlight_overlay 双轮渐入渐出）
- 防御页「来源」按钮 → `_on_source_clicked(nav_key, seq_label)` → `_do_highlight_in_source()` → 综合填写行高亮



## 九点六、防御公式与抗性机制（v1.0 2026-06-17）

### 新防御公式
```
敌方基础防御 = 792 + 8 × 敌人等级
敌方最终防御 = 敌方基础防御 × (1 − 无视防御) × (1 − 忽视/减少防御)
防御乘区 = (800 + 8 × 角色等级) / (敌方最终防御 + 800 + 8 × 角色等级)
```
`calc_defense_zone(char_level, enemy_level, total_ignore, total_reduce)` — `total_reduce` 默认 0.0 向后兼容。

### 新抗性公式
```
最终抗性 = 基础抗性 × (1 + 抗性提升/100) − 抗性减少 − 外部减免
抗性乘区 = 1 − 最终抗性/100  (最终抗性 ≥ 0)
抗性乘区 = 1 − 最终抗性/200  (最终抗性 < 0，负抗性收益减半)
```
- `calc_resistance_zone` 实现
- `get_resistance_multiplier` 改用 `_res_mult` 字典缓存（随 `_recalc` 同步写入，避免 QTableWidget 时序问题）

### 序列号统一规范
所有需要定位的词条统一使用综合填写 `collect_data()` 返回的第 5 字段 `item_data[4]`（格式 `"常驻N"/"触发N"`）：
- EnemyDefensePage.recalc
- EnemyResistancePage._recalc
- 反向跳转 3 处：`_do_highlight_in_source` / `_highlight_in_source` 使用 `pw.collect_data()[r][4]` 匹配
